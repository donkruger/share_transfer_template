# Submission Workflow

The Submission Workflow is the final step in the share transfer process. It provides a review interface, declaration acceptance, multi-format export (PDF, CSV, share transfer CSV), and email delivery.

## Key Source Files

| File | Role |
|------|------|
| `app/pages/3_Submit.py` | Submit page UI, review, declaration, and submission orchestration |
| `app/components/submission.py` | `handle_search_results_submission()`, `handle_portfolio_submission()` |
| `app/components/feedback.py` | `render_feedback_component()` -- optional feedback collection |
| `app/email_sender.py` | SMTP email delivery with attachments |
| `app/pdf_generator.py` | PDF report generation via ReportLab |

## Submission Flow

### Pre-Submission Validation

The page enforces two hard gates before any submission UI is rendered:

1. **User information** -- `user_name` and `user_id` must be set in session state. If missing, the user is redirected to the main page.
2. **Instrument selection** -- At least one instrument must be selected via `SelectionManager.get_selections()`. If empty, the user is redirected to search.

### Review Interface

#### Portfolio Summary

Metric cards display:
- Total instrument count
- Number of unique exchanges
- Number of distinct asset types
- Selection period (date of earliest selection)

#### Configuration Status

- **Complete** -- If `PortfolioService.is_portfolio_complete()` returns true, a success badge and optional portfolio summary table (as a DataFrame) are shown.
- **Incomplete** -- A warning with remaining count and a link to the Portfolio page.

#### Instrument Detail Cards

Each selected instrument is rendered in an expandable card showing:
- Name, ticker, ISIN, asset type
- Exchange, currency, relevance score, match type
- Selection metadata (timestamp, source query)
- Wallet availability via `WalletFilterEngine.get_available_wallets()`
- Individual remove button

### Declaration

A checkbox declaration reads: "I declare that the information provided is accurate and I understand that this submission is for informational purposes to help me find available instruments in the EasyEquities ecosystem."

The submit button is disabled until the declaration is accepted.

### Submission Handling

Two submission paths exist depending on portfolio configuration status:

#### Complete Portfolio Path (`handle_portfolio_submission`)

Generates:
1. **Share Transfer CSV** -- Broker-specific format via `PortfolioService.generate_share_transfer_data()` with exact column mapping (SX/EE, User ID, TrustAccountID, ShareCode, InstrumentID, Qty, Base Cost, Excel Date, SettlementDate, Last Price, BrokerID_From, BrokerID_To, Reference).
2. **PDF Report** -- Generated by `app/pdf_generator.py` using ReportLab with section-by-section rendering and automatic page breaks.
3. **Standard CSV** -- Instrument data in a general-purpose format.

#### Basic Path (`handle_search_results_submission`)

Used when portfolio configuration is not complete. Sends instrument data without share transfer details.

### Email Delivery

`send_search_results_email_direct()` in `app/email_sender.py` handles the email:

- **SMTP configuration** -- Credentials loaded from `st.secrets["email"]` (sender_email, sender_password, smtp_server, smtp_port).
- **Recipient** -- Configurable; defaults to the operations inbox.
- **Body** -- Plain text with user details, wallet context, instrument list, optional submission notes, and optional feedback.
- **Attachments** -- PDF report, standard CSV, and (if portfolio is complete) share transfer CSV.

### Post-Submission

After a successful submission:
1. `SelectionManager.clear_selections(confirm=True)` clears all instrument selections.
2. `PortfolioService.clear_portfolio_data()` resets portfolio state.
3. A success message with balloons animation is displayed, along with a visible summary of next steps (email confirmation, team review, downloadable reports).
4. Navigation buttons offer "Start New Search" or "Get More AI Help".

## Optional Components

### Feedback Collection

`render_feedback_component()` provides:
- Entity name field
- Email field
- Category selector
- Satisfaction rating (1-5)
- Free-text message

Feedback data is included in the submission email body under a dedicated section.

### Submission Notes

A free-text area allows the user to add contextual notes (e.g., "Looking to diversify my portfolio with these instruments"). Notes are included in the email body.

### Search History

An expandable section at the bottom of the page displays the user's recent search history (last 5 searches) with timestamps, queries, result counts, and wallet context.
